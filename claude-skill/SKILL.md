# 报销助手 Skill

将一堆发票图片/PDF 自动识别、分类、配对、整理，并生成 Excel 报销统计报表。

## 工作流程

```
用户提供发票文件夹 → Claude 逐一识别 → 自动分类 → 智能配对 → 整理到目录 → 生成 Excel 报表
```

## 第一步：确认输入

Ask the user for:
1. **输入目录**: 发票文件所在的文件夹路径（支持 jpg/jpeg/png/bmp/webp/pdf）
2. **输出目录**: 整理结果的输出路径（默认在输入目录同级创建 `报销结果_YYYYMMDD/`）
3. **操作模式**: 复制还是移动原始文件（默认复制，保留原文件）

## 第二步：扫描和识别

1. 递归扫描输入目录中所有支持的文件（jpg/jpeg/png/bmp/webp/pdf）
2. 逐一用 Read 工具查看每个文件（Claude 是多模态的，可以直接看图片和 PDF）
3. 对每个文件，提取以下信息并记录到一个 JSON 结构中：

```json
{
  "file_path": "原始文件路径",
  "type": "taxi|train|flight|hotel|meal|other",
  "subtype": "具体平台，如滴滴出行、12306、XX航空",
  "amount": 123.45,
  "date": "YYYY-MM-DD (开票日期)",
  "service_date": "YYYY-MM-DD (实际消费日期，比开票日期更重要)",
  "merchant": "商家名称",
  "invoice_number": "发票号码",
  "order_number": "订单号/交易号",
  "is_invoice": true,
  "description": "简要描述"
}
```

### 分类规则

| type | 类别名 | 识别范围 |
|------|--------|---------|
| taxi | 打车票 | 滴滴、高德、美团打车、曹操、首汽、出租车、网约车 |
| train | 火车飞机票 | 12306、火车票、高铁、动车 |
| flight | 火车飞机票 | 各航空公司、机票、登机牌 |
| hotel | 住宿费 | 酒店、宾馆、民宿 |
| meal | 餐费 | 餐厅、外卖、美团、饿了么 |
| other | 其他 | 无法识别的 |

### is_invoice 判断

- `true`: 正式发票（有发票代码、发票号码、税额等）
- `false`: 行程单、收据、凭证、水单等非正式发票

### 日期提取要点

- `service_date`（实际消费日期）比 `date`（开票日期）更重要
- 打车：乘车时间；火车/飞机：出发日期；酒店：入住日期；餐饮：消费日期
- 发票可能后补开，开票日期可能晚于消费日期

## 第三步：智能配对

打车、住宿等场景经常有「凭证/行程单 + 发票」两份文件，需要智能配对：

配对规则（至少满足 2 个条件才配对）：
1. **订单号匹配**（最高优先级）：订单号完全相同 → 直接配对
2. **平台/商家匹配**：同一平台（如都是"滴滴出行"）
3. **日期匹配**：实际消费日期相同或相差 ±1 天
4. **金额匹配**：金额相同或差异在 ±5% 以内
5. **类型匹配**：同一 type

## 第四步：整理文件

将文件复制/移动到输出目录，结构如下：

```
报销结果_20240114/
├── 打车票/
│   └── 2024-01-15_滴滴出行_35.00元/
│       ├── 01_2024-01-15_凭证_滴滴出行_35.00元.jpg
│       └── 02_2024-01-15_发票_滴滴出行_35.00元.pdf
├── 火车飞机票/
│   └── 2024-01-10_12306_263.00元/
│       └── 2024-01-10_发票_12306_263.00元.pdf
├── 住宿费/
├── 餐费/
├── 其他/
├── 待确认/  (无法确定消费日期的独立发票)
└── 报销统计.xlsx
```

文件夹命名：`{消费日期}_{商家}_{金额}元`
文件命名：`[序号_]{消费日期}_{发票|凭证}_{商家}_{金额}元.{ext}`

## 第五步：生成 Excel 报表

运行 `scripts/generate_report.py` 脚本生成 Excel 报表。

用法：
```bash
python <skill-path>/scripts/generate_report.py <output_dir> <invoice_data_json_path>
```

其中 `invoice_data_json_path` 是一个 JSON 文件，包含所有已整理的发票信息数组。格式：
```json
[
  {
    "file_path": "整理后的文件路径",
    "type": "taxi",
    "category_name": "打车票",
    "amount": 35.00,
    "date": "2024-01-15",
    "service_date": "2024-01-15",
    "merchant": "滴滴出行",
    "subtype": "滴滴出行",
    "invoice_number": "12345678",
    "is_invoice": true,
    "description": "北京-朝阳"
  }
]
```

报表包含：
- **汇总表**：各类别发票数量、金额小计、总金额
- **分类明细表**：每个类别一个 sheet，包含序号、日期、商家、金额、发票号码、描述

## 第六步：汇总展示

处理完成后，在终端展示汇总信息：

```
==================================================
处理完成！汇总如下：
==================================================
  打车票: 5 张, ¥385.50
  火车飞机票: 2 张, ¥526.00
  住宿费: 3 张, ¥1,245.00
  餐费: 4 张, ¥312.80
--------------------------------------------------
  总计: ¥2,469.30
==================================================

文件已整理到: /path/to/报销结果_20240114
统计报表: /path/to/报销结果_20240114/报销统计.xlsx
```

## 重要注意事项

- 金额提取实际支付金额，不是税额
- 日期统一转换为 YYYY-MM-DD 格式
- 配对组的文件夹日期以凭证/行程单的实际消费日期为准
- 只有正式发票（is_invoice=true）才计入报表金额
- 如果发票数量较多（>20），建议分批处理并给用户进度反馈
- 生成报表前先 `pip install openpyxl`（如未安装）
